Homework 1: Tasks for oj36

Your personal task is to develop a system for quizzes and their contestants.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Contestant. With the following private fields:

  1. String name
  2. String profession
  3. String skill
  4. int wins
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Quiz. With the following private fields:

  1. int id
  2. String name
  3. String genre
  4. List<Contestant> contestants
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Quiz> quizzes in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one quiz with two contestants. [4 marks]

Create controller co2103.hw1.controller.QuizController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/quizzes" should show view quizzes/list. [2 marks]
    • Add the list of all quizzes from co2103.hw1.Hw1Application to the model as attribute quizzes. [2 marks]
    • Create the corresponding JSP to list the name and genre of each quiz followed by a link to URL "/contestants?quiz={id of quiz}", e.g., "/contestants?quiz=3". [4 marks]
  2. Requests for URL "/newQuiz" should show view quizzes/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="quiz" and action="/addQuiz" and inputs for all fields of class Quiz except for contestants. [5 marks]
  3. POST requests for URL "/addQuiz" should add a new quiz to the list of all quizzes in co2103.hw1.Hw1Application and redirect to /quizzes. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.ContestantController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/contestants" should show view contestants/list. [2 marks]
    • Based on request parameter quiz determine the corresponding quiz from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all contestants of that quiz to the model as attribute contestants. [2 marks]
    • Create the corresponding JSP for the view to list the name, profession, skill, and wins of each contestant. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newContestant?quiz={id of quiz whose contestants are listed}", e.g., "/newContestant?quiz=3". [4 marks]
  2. Requests for URL "/newContestant" should show view contestants/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="contestant" and action="/addContestant" and inputs for all fields of class Contestant. [6 marks]
    • Also add the request parameter quiz with the ID of the current quiz. [4 marks]
  3. POST requests for URL "/addContestant" should add a new contestant to the list of contestants of the quiz identified by request parameter quiz and then redirect to /quizzes. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Quiz and make it available in controller QuizController. [4 marks]

When adding a new quiz validate that

  1. The value of field id is unique within the list Hw1Application.quizzes. [3 marks]
  2. The value of field name is not empty. [2 marks]
  3. The value of field genre is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Contestant and make it available in controller ContestantController. [4 marks]

When adding a new contestant validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field profession is not empty. [2 marks]
  3. The value of field skill is either "Sport", "Literature", or "Science". [3 marks]
  4. The value of field wins is between (including) 0 and 30. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
